## Video Production Philosophy

Not every lesson needs a video. Text-based learning works well for:
- Detailed code walkthroughs (students can copy/paste)
- Reference material (students revisit often)
- Dense technical specifications

Video excels for:
- Conceptual explanations (visual diagrams help)
- Process demonstrations (Event Storming, modeling)
- Motivation and engagement (course intros, key concepts)
- Complex workflows (multi-step processes)

## Update Schedule by Subscriber Count

### 0-50 Subscribers (Launch Phase)

**Focus:** Phase 1 videos only
**Update Frequency:** Weekly written content updates
**Video Production:** Complete Phase 1 before accepting students

```
Week 1-2: Course intro + Module 1 videos (4 videos)
Week 3-4: Module 2 + 4 + 5 concept videos (4 videos)
Week 5: Review, polish, soft launch
```

**Rationale:** Minimal viable video content. Test market fit before heavy investment.

---

### 50-200 Subscribers (Early Growth)

**Focus:** Complete Phase 2
**Update Frequency:** Bi-weekly content updates + monthly video releases
**Video Production:** 1-2 videos per month

```
Month 1: Module 3 videos (Entities, Aggregates)
Month 2: Module 6 AI videos (high value differentiator)
Month 3: Module intros + polish existing videos
```

**Triggers to start Phase 2:**
- Course completion rate > 40%
- Positive student feedback on Phase 1 videos
- Revenue covers production costs

### 200-500 Subscribers (Established Course)

**Focus:** Begin Phase 3

**Video Production:** Prioritize based on student feedback

## Reusable Animation Components

Invest time upfront in Remotion components that can be reused:

### High-Value Components (Build First)

1. **BoundedContextDiagram** - Animated context boxes with labels
2. **StickyNoteBoard** - Event Storming visualization
3. **DataFlowDiagram** - Arrows showing read/write paths
4. **Timeline** - Event sourcing event sequence
5. **CodeHighlight** - Syntax highlighted code blocks
6. **ComparisonSplit** - Before/after or good/bad patterns
